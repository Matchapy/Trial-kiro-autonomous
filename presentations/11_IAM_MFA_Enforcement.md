# IAM MFA 強制適用

## サービスカテゴリ
**セキュリティ**

---

## サービス概要
すべてのアカウントタイプのルートユーザーに対してMFA（多要素認証）を強制適用する画期的なセキュリティ強化。パスワード関連攻撃の99%以上を防止します。

---

## 主な機能と特徴

### 1. 全面的なMFA適用
- すべてのアカウントタイプでMFA必須化
- ルートユーザーの100%カバレッジ達成
- ハイパースケールクラウドプロバイダーとして業界初

### 2. 複数のMFA方式サポート
- FIDO認証セキュリティキー
- FIDO2パスキー
- 仮想MFAデバイス
- ハードウェアMFAデバイス

### 3. 柔軟なデバイス管理
- ルートユーザーおよびIAMユーザーごとに最大8つのMFAデバイス登録可能
- 複数デバイスによる冗長性
- デバイス紛失時のバックアップ

---

## MFA強制適用の進化

### セキュリティジャーニー

#### 2024年5月
AWS Organizations管理アカウントのルートユーザーに対してMFA要求開始

#### 2024年6月
スタンドアロンアカウントのルートユーザーにMFA要求を拡大

#### 2024年11月
AWS Organizationsの中央集権的ルートアクセス管理を導入

#### 2025年6月
すべてのアカウントタイプのルートユーザーに対してMFA強制適用を完了

## サポートされるMFA方式

### 1. FIDO認証セキュリティキー
**特徴**:
- 物理的なセキュリティキー
- フィッシング耐性
- 高いセキュリティレベル

**推奨ユースケース**:
- 高セキュリティ要件
- 企業環境
- コンプライアンス対応

### 2. FIDO2パスキー
**特徴**:
- ユーザーフレンドリー
- パスワードレス認証
- デバイスに紐付け

**推奨ユースケース**:
- 日常的なアクセス
- 利便性重視
- モバイルデバイス使用

### 3. 仮想MFAデバイス
**特徴**:
- スマートフォンアプリ使用
- 追加ハードウェア不要
- 時間ベースのワンタイムパスワード（TOTP）

**推奨アプリ**:
- Google Authenticator
- Microsoft Authenticator
- Authy

### 4. ハードウェアMFAデバイス
**特徴**:
- 専用物理デバイス
- 高い信頼性
- ネットワーク接続不要

**推奨ユースケース**:
- 最高レベルのセキュリティ
- インターネット接続制限環境
- 長期使用

## 技術仕様

### 認証フロー
```
1. ユーザー名とパスワード入力
2. MFAチャレンジ提示
3. MFAデバイスによる認証
4. アクセス許可
```

### サポート機能
- 複数MFAデバイスの登録
- デバイスの有効化/無効化
- 紛失デバイスのリカバリー
- 信頼できるデバイスの記憶（オプション）

### 統合
- AWS Management Console
- AWS CLI
- AWS SDK
- AWS API

## 導入ステップ

### 個人アカウント

#### Step 1: MFAデバイスの選択
1. セキュリティ要件の評価
2. 適切なMFA方式の選択
3. デバイスの準備

#### Step 2: MFAの有効化
1. IAMコンソールにアクセス
2. セキュリティ認証情報を選択
3. MFAデバイスを割り当て
4. デバイスの設定と検証

#### Step 3: バックアップMFAの設定
1. 追加MFAデバイスの登録
2. バックアップコードの保存
3. リカバリープロセスの確認

3. 展開計画の作成

#### Step 2: 段階的展開
1. パイロットグループでのテスト
2. フィードバックの収集
3. 組織全体への展開

#### Step 3: トレーニングとサポート
1. ユーザートレーニングの実施
2. サポート体制の確立
3. ドキュメントの提供

## トラブルシューティング

### デバイス紛失時
1. バックアップMFAデバイスの使用
2. AWSサポートへの連絡
3. アカウントリカバリープロセスの実行

### デバイスが動作しない
1. 時刻同期の確認（TOTP）
2. バッテリー確認（ハードウェアデバイス）
3. 別の登録済みデバイスの使用

### アクセス不可
1. アカウントリカバリープロセス
2. AWSサポートへの連絡
3. 本人確認手続き

## まとめ

IAM MFA強制適用は、AWS セキュリティの大きなマイルストーンです。ハイパースケールクラウドプロバイダーとして初めて、すべてのルートアカウントで100% MFAカバレッジを達成しました。

パスワード関連攻撃の99%以上を防止し、FIDO認証キーやFIDO2パスキーなど、最新のユーザーフレンドリーなMFA技術をサポートしています。最大8つのMFAデバイスを登録できる柔軟性により、デバイス紛失時のリスクも軽減されます。

組織の規模や業界を問わず、すべてのAWSユーザーがこの強力なセキュリティ機能の恩恵を受けることができます。MFAは、クラウドセキュリティの基本であり、AWSはその実装において業界をリードしています。
