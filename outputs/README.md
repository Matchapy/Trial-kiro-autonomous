# Output Files Directory

This directory contains all outputs generated by the AWS re:Invent 2025 Research Automation.

## Directory Structure

```
outputs/
├── presentations/          # PowerPoint presentations
│   └── AWS_reInvent_2025_Services.pptx
├── screenshots/           # AWS Console screenshots
│   ├── service_name_main.png
│   └── service_name_pricing.png
├── data/                  # JSON and text data
│   ├── announcements.json
│   ├── research_results.json
│   └── summary_report.txt
└── automation.log        # Execution logs
```

## File Descriptions

### Presentations

- **AWS_reInvent_2025_Services.pptx**
  - Complete PowerPoint presentation
  - Contains all researched services
  - Includes screenshots, pricing, examples
  - Ready to share with engineering teams

### Screenshots

- **[service]_main.png** - Main console/marketing page
- **[service]_pricing.png** - Pricing information page
- High resolution (1920x1080)
- Organized by service name

### Data Files

- **announcements.json**
  - Raw announcements extracted from blog
  - Contains titles, descriptions, links
  - Timestamp of extraction

- **research_results.json**
  - Comprehensive research data
  - Service overviews, features, benefits
  - Pricing information, use cases
  - Documentation URLs

- **summary_report.txt**
  - Human-readable summary
  - Quick overview of all services
  - Key information at a glance

### Logs

- **automation.log**
  - Detailed execution logs
  - Debug information
  - Error messages and warnings
  - Timestamps for all operations

## Usage

After running the automation:

```bash
# View presentation
open presentations/AWS_reInvent_2025_Services.pptx

# View summary
cat data/summary_report.txt

# Check logs
tail -f automation.log

# View research data
cat data/research_results.json | python -m json.tool
```

## File Sizes

Typical file sizes:
- PowerPoint: 300-500KB (depends on screenshots)
- Screenshots: 150-200KB each
- JSON files: 1-10KB
- Log file: 5-50KB

## Archiving

To archive outputs:

```bash
# Create archive
tar -czf reinvent-$(date +%Y%m%d).tar.gz presentations/ data/ screenshots/

# Upload to S3
aws s3 cp reinvent-$(date +%Y%m%d).tar.gz s3://your-bucket/archives/
```

## Cleanup

To clean old files:

```bash
# Clean all outputs
cd .. && make clean

# Or manually
rm -f presentations/*.pptx data/*.json data/*.txt screenshots/*.png
```

## Notes

- Files are overwritten on each run
- Archive important presentations before re-running
- Log file is appended (not overwritten)
- Directory structure is preserved by .gitkeep files
